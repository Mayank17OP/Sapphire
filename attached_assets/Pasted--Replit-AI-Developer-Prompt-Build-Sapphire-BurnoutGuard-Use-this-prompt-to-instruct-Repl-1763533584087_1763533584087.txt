## Feature & Component Requirements (must implement)

### Visual & UI components

* **Unique design language** across app — do NOT copy Builder.io themes; build custom CSS/Tailwind tokens.
* **Glass-morphism cards** using `backdrop-filter: blur(...)` with soft borders and inner glow.
* **Subtle particle background** on hero sections using `tsparticles` or `particles.js` (configurable intensity).
* **Custom cursor effects**: a subtle ring that follows pointer; clickable elements show a pulse.
* **Animated "fatigue level" indicator**: circular or bar meter that changes color from cyan -> lavender -> deep red based on level. It must animate smoothly.
* **Subtle particle and micro-interaction animations** (hover, focus) with Framer Motion.

### Landing Page (`/`)

* **Hero** with animated "Focus Energy" visualization — can be an animated SVG canvas or small interactive component that pulses with mock data.
* **Webcam demo** showing real-time fatigue detection: if webcam access allowed, show `getUserMedia` feed in a rounded glass card and run a mock detection algorithm (blink/eye closure counter simulated, not full ML): e.g., use average brightness + timed blink simulation to produce values. If webcam denied, show fallback demo video or animated placeholder.
* Prominent value proposition: **"Stop burnout before it starts"** with download CTA (download button triggers a modal that explains desktop app availability and simulates a file download link).

### Dashboard (`/dashboard`)

* **Live "Focus Energy" graph**: use Recharts and feed **mock** data that updates every second (use `setInterval` to push new points and keep a sliding window). Include play/pause controls.
* **Webcam feed placeholder** with `Start/Stop detection` buttons and visual indicator of detection state.
* **Fatigue metrics panel**: show metrics such as `Eye Closure (avg)`, `Yawning Count`, `Posture Alert`, `Focus Score`. Values should update (mock) and be highlighted when thresholds are crossed.
* **Break suggestion cards** that appear as chips/cards when thresholds are hit; clicking a suggestion opens the Interventions page anchored to the recommended exercise.

### Chat (`/chat`)

* **Chat UI** with an animated Sapphire AI avatar (SVG + subtle glow), messages area, input area, and pre-set conversation starters.
* Implement **real-time typing simulation** for bot responses: animate a typing indicator for 500–1500ms before rendering full message; also support streaming-like reveal (character-by-character) for realism.
* **Mood tracking**: emoji selector (range from 😃 to 😔) that stores selection in `localStorage` and updates a small mood history.

### Interventions (`/breaks`)

* **Interactive breathing exercise**: animated box-breathing component (4-4-4-4 or 4-4-4) with circle/box that expands/contracts and textual cues.
* **20-20-20 rule timer**: when started, counts down a 20-minute work timer (or allow toggling between a short demo 20s for quick testing) then triggers a 20-second countdown for looking at distant object; show progress and a big dismiss button.
* **Quick stretch animations**: small Lottie-like animations or CSS animated SVGs with play/pause controls.
* **Focus Session timer**: Pomodoro (25/5 preset) with start/pause/reset. When session completes, show a completion modal with the option to log mood or take a break.

### Settings (`/settings`)

* **Privacy controls** with webcam permission toggle (this only toggles whether the app will attempt getUserMedia; obviously explain in UI that the webcam feed never leaves the browser).
* **Notification preferences** (desktop notifications mock; show permission request flow if allowed by browser; fallback to toast messages).
* **Sensitivity sliders** for fatigue detection thresholds (eye closure sensitivity, yawn detection sensitivity). Sliders must persist to `localStorage`.
* **Account management** area (mock form to update name/email) — forms are submittable and show loading states, and persist to `localStorage`.

### Other Technical Requirements

* Use `localStorage` to persist at least: theme preference (light/dark), fatigue sensitivity settings, mood history, and notification preference.
* All interactive elements must have hover/focus states, and show a loading state for async actions (simulate 700–1200ms delays).
* Mobile responsive layout with breakpoints; primary UX optimized for desktop.
* Provide keyboard-accessible controls and ARIA attributes for major components.
* Provide inline comments in code for complex parts and a README with instructions to run locally on Replit.

## Implementation Guidance (explicit suggestions for the generator)

* Use **Vite + React** starter template. Use Tailwind with `postcss` for quickly composing the glass effect tokens. Add a tiny custom CSS file for glow and cursor effects.
* For mock webcam fatigue detection, **DO NOT** attempt to ship a real ML model. Instead implement a deterministic simulated detector that:

  * Monitors blink rate by counting quick changes in brightness on the webcam feed (approximation) OR uses a simulated algorithm that increases fatigue over time when user hasn't taken breaks.
  * Exposes a `startDetection()` / `stopDetection()` API that updates app state with `eyeClosure`, `yawns`, `posture` flags.
* Use `tsparticles` for hero particles and small parallax effect.
* Use `Recharts` for the Focus Energy chart and animate updates smoothly.
* Use `Framer Motion` for page transitions, card hover lifts, and small micro-interactions.
* Keep the code modular: components like `FatigueMeter`, `WebcamDemo`, `FocusChart`, `BreathingGuide`, `ChatBot`, `PomodoroTimer`, `SettingsPanel`.
